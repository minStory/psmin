추상클래스 : 미완성 설계도
선언부 : 함수를 호출하는 부분을 공통으로 만든다 -> 추상클래스(부모);
구현부 : 호출했을 때 처리할 내용-> 추상클래스를 상속받은 자식이 완성한다.
1) 미완성 설계도, 미완성 메서드
2) 다른 클래스에 도움을 준다(상속 받은 자식) -> 인스턴스 생성불가 -> 몸통이 없다
3) 인스턴스 생성을 위해서는 자식이 상속을 통해 추상 메서드 완성
4) 추상클래스는 추상메서드를 구현한다는 방법만 다르고 일반 클래스와 동일하다.
	abstract class Player()
		1. 일반 인스턴스 메서드도 만들고 상속방을 수 있다.	
		2. 생성자도 만들 수 있다.(오버로딩) ->슈퍼생성자 만들 수 있다.
		3. 추상 메서드를 만들면 상속받은 자식은 무조건 추상메서드를 구현해야한다.(오류)
