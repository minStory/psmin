01 정보보안

정보보안
: 기업의 정보에 대해서 허가되지 않은 접근, 변경, 삭제 등으로부터 보호하는 것

정보보안 요소
	1) 기밀성(Confidentiality) ::암호
		- 인가된 사용자만 정보 자산에 접근할 수 있도록 한다.
	2) 무결성(Integrity) ::인증, 인가
		- 인증된 사용자가 인가된 방법으로만 정보를 변경 및 접근할 수 있다.
	3) 가용성(Availability) ::고가용성
		- 원하는 시점에 언제든 정보 자산에 접근이 가능하도록 한다.
	4) 인증(Authentication)
		- 접속한 사용자가 허가받은 사용자인지 확인하는 것
	5) 부인방지(Non-Repudiation)
		- 정보를 보냈다는 것을 발뺌하지 못하도록 하는 것

AAA(Authentication, Authorization, Accounting)
	1) 인증(Authentication)
		- 사용자의 신원을 검증
	2) 권한부여(Authorization)
		- 인증된 사용자에게 권한과 서비스 부여
	3) 계정관리(Accounting)
		- 사용자의 자원 사용 정보를 모아서 관리

정보보안 거버넌스(Information Security Governance)
: 정보의 무결성, 서비스의 연속성, 정보자산의 보호를 위한 것

정보보안 거버넌스의 3요소
	1) 데이터 무결성 ::무결성
	2) 서비스 연속성 ::가용성
	3) 정보자산의 보호 ::기밀성

인증제도
	1) ISMS(정보보호 관리체계 인증)
		- 기술적, 물리적, 관리적 등 종합적인 정보보호 관리체계에 대한 인증제도
	2) PIMS(개인정보보호 관리체계)
		- 기관 및 기업이 체계적.지속적으로 보호 업무를 수행하는지
		  객관적으로 심사 후 인증 부여
	3) ISMS-P(정보보호 및 개인정보보호 관리체계 인증) ::ISMS + PIMS
		- 정보보호 및 개인정보보호를 위한 조치와 활동이 기준에 적합함을
		  인터넷진흥원 또는 인증기관이 증명하는 제도
	4) ITSEC ::유럽
		- 영국, 프랑스, 독일, 네덜란드 주축의 보안성 평가 기준서
	5) TCSEC ::미국
		- 미국의 신뢰성 있는 컴퓨터 시스템 평가기준
	6) CC ::유럽 + 미국 통합
		- 국가마다 상이한 평가 기준 연동, 결과를 상호 인증하는 국제 기준

Secure SDLC
: SDLC에 보안 강화를 위한 프로세스를 포함한 것

Secure SDLC 방법론
	1) CLASP : SDLC의 초기 단계에서 보안 강화 적용 -> 초기
	2) MS-SDL : MS사가 개발한 SDLC를 개선한 방법론 -> MS사, SDLC
	3) Seven Touchpoints : 보안의 모범사례를 SDLC에 통합한 방법론 -> 모범사례

시큐어 코딩 ::KISA -> 우리나라 기준
	1) OWASP
		- 오픈소스 웹 애플리케이션 보안 프로젝트
		- OWASP Top 10 
시큐어 코딩 가이드
	1) 입력 데이터 검증 및 표현
		- 입력값에 대해 발생하는 보안 약점
		- SQL Injection, XSS, 자원 삽입, 위험한 형식 파일 업로드,
		  명령 삽입, 메모리 버퍼 오버플로
	2) 보안기능
		- 보안 기능을 부적절하게 구현하는 경우 발생하는 보안 약점
		- 적절한 인증 없이 중요기능 허용, 부적절한 인가,
		  취약한 암호화 알고리즘 사용, 하드코딩된 패스워드,
		  패스워드 평문 저장, 취약한 패스워드 허용
	3) 시간 및 상태
		- 병행되는 시스템 환경에서 시간 및 상태를 부적절하게 관리하여 발생하는 보안 약점
		- 경쟁 조건, 종료되지 않는 반복문 또는 재귀 함수
	4) 에러 처리
		- 에러를 처리하지 않거나 에러 정보에 중요 정보가 포함될 때 발생하는 보안 약점
		- 오류 메시지 정보 노출, 오류 상황 대응 부재, 부적절한 예외 처리
	5) 코드 오류
		- 개발자가 범할 수 있는 코딩 오류로 유발되는 보안 약점
		- 널 포인터 역참조, 부적절한 자원 해제, 해제된 자원 사용,
		  초기화되지 않은 변수 사용
	6) 캡슐화
		- 중요한 데이터나 기능을 불충분하게 캡슐화하거나 잘못 사용해 발생하는 보안 약점
		- 잘못된 세션에 의한 정보 노출, 제거되지 않은 디버그 코드,
		  시스템 정보 노출, 잘못된 접근 지정자
	7) API 오용
		- 의도된 사용에 반하는 방법으로 API를 사용하거나
		  보안에 취약한 API를 사용하여 발생하는 보안 약점
		- DNS에 의존한 보안 결정
		- 취약한 API 사용

백업과 복구
	1) 재난 복구 전략 시 지표
		- RP : 복구까지 걸린 시간
		- RTO : 시스템 장애 시 허용되는 최대 시간
		- RPO : 손실을 허용되는 데이터의 양
		- MTD : 비즈니스 연속성 관리 지표, 장애 시 허용되는 최대 시간
	2) 백업과 복구를 위한 전략
		- 풀-이미지 백업 : 전체 백업
		- 차등 백업 : 풀백업 이후부터 발생한 모든 변경 사항을 백업
		- 증분 백업 : 풀백업 이후의 그날 그날 점차적으로 백업
		- 실시간 백업 : 지속적인 백업, 즉각적으로 모든 변경사항을 백업
		- 합성 백업 : 풀백업 + 증분백업

암호 알고리즘

대칭키 암호(Symmetric Key)
: 암호화할 때와 복호화할 때의 키가 동일한 암호 시스템, 혼돈과 확산의 성질을 이용
	1) 블록암호 알고리즘
		- DES : 64bit 블록, 56bit 암호화 키, 16Round, Feistel 방식
		- 3-DES : DES와 동일, 암호화 키 2개 사용
		- AES : 128bit 평문을 128/192/256bit로 암호화, 10/12/14Round, SPN 방식
		- SEED : 국내 개발, 128bit 및 256bit
		- ARIA : 국내 개발, AES와 똑같이 128/192/256bit 지원
		- IDEA : 스위스 개발, 64bit 블록, 128bit 암호화 키, 8Round
		- SKIPJACK : 미국 개발, Clipper 칩, 64bit 블록, 80bit 암호화 키, 32Round 
	2) 스트림암호 알고리즘
		- LFSR : 경량 및 고속 동작
		- RC4 : 로널드 라이베스트
		- A5 : GSM 휴대폰 체계

비대칭키 암호
: 암호화와 복호화에 이용하는 키가 다른 방식, 공개키 암호 방식이라고도 한다

비대칭 키의 종류
	1) 공개키(Public Key) : 대중에게 공개된 키
	2) 개인키(Private Key) : 개인이 가지고 있으면서 관리하는 키

비대칭 키 알고리즘
	1) 소인수 분해 기반
		- RSA : 대표적인 공개키 알고리즘
		- Robin : Robin이 개발, RSA보다 빠름
	2) 이산대수 기반
		- Diffie-Hellman : 키관리 센터 없이 공개키 전달 가능
		- DSA : 미국 전자서명 표준
		- ELGamal :  같은 평문에서 다른 암호문 생성
		- KCDSA : KISA에서 개발
	3) 타원 곡선
		- ECC : 타원 곡선상의 이산대수 이용

전자서명
: 인증서 형태로 발급되는 자신만의 디지털 인감 도장, 안전한 디지털 서명

단방향 암호화
: Hash를 이용하여 암호화, 복호화는 불가능
	1) 해시 함수 특성
		- 역상 저항성 : 해시값을 생성하는 입력값을 알아낼 수 없다
		- 제2 역상 저항성 : 동일한 해시값을 생성하는 다른 입력값을  알아낼 수 없다
		- 충돌 저항성 : 해시값이 같은 두 입력값을 알아낼 수 없다
	2) 해시 함수 종류
		- MD5 ::사용 안함
		- SHA ::주로 사용
		- HAS-160 ::국내꺼
	3) 암호학적 해시 함수의 결점
		ㄱ. 무차별 대입 공력(Brute-Force Attack)
			- 해시 함수는 빠르기 때문에 무차별적 데이터를 넣다보면
			  암호화가 깨질 수 있다
		ㄴ. Rainbow Table 공격
			- 암호 유형을 정의한 Rainbow Table을 만들어 하나씩 대입해보면
			  암호를 찾을 수 있다
	4) 암호학적 해시 함수의 보완
		ㄱ. 키 스트레칭(Key Stretching)
			- 무차별 대입 공격을 방지
			- 해시 암호화를 여러번 반복
		ㄴ. 솔팅(Salting)
			- Rainbow Table 공격 방지
			- 데이터 앞/뒤에 임의의 값을 넣어 해시값을 생성

전자우편 보안
	1) 암호화 프로토콜
		- PGP : 전자우편 보안의 표준
		- PEM : 특정 키가 있어야만 내용을 볼 수 있다
		- S/MIME : 첨부물에 대한 보안
		- DKIM : 발신자가 발송 정보를 위장할 수 없다

코드 오류
	1) 코드의 유형
		- 순차 코드(Sequence Code) ::일정한순서(학교의 반번호)
		- 블록 코드(Block Code) ::별도의 집단(시, 군, 구)
		- 10진 코드(Decimal Code) ::10진수의 형태
		- 그룹 분류 코드(Group Classification Code) ::대분류, 중분류, 소분류
		- 연상 코드(Mnemonic Code) ::어떤 대상인지 쉽게 연상 가능
		- 표의 숫자 코드(Significant Digit Code) ::물리적 수치(길이, 너비)
		- 합성 코드(combined Code) ::복합적인 코드
	2) 코드의 오류 발생 형태
		- 생략 오류 ::빠뜨린 것
		- 필사 오류 ::잘못 쓴 것
		- 전위 오류 ::바뀐 것
		- 이중 오류 ::전위오류가 2개
		- 추가 오류 ::하나 더 쓴 것
		- 임의 오류 ::복합적으로 나타난 것

인증(Authenticaion)
: 사용자의 정보를 확인하고 접근 권한을 검증하는 보안 절차
	1) 인증 유형
		- 지식 기반 인증(ID, PW)
		- 소유 기반 인증(신분증, 여권)
		- 생체 기반 인증(지문, 홍채)
		- 행위 기반 인증(서명, 동작)
		- 위치 기반 인증(IP, GPS)

인가(Authorization)
: 로그인 후, 인증된 사용자에게 권한을 부여

인증 방식
	- 계정 정보를 요청 헤더에 넣는 방식
	- Cookie/Session 방식
	- 토큰 기반 인증 방식 ::여러 SNS의 토큰을 받아 한번에 게시글 작성
	- SSO(Single Sign-On) ::한번 로그인하면 여러군데에서 같이 로그인 됨
	- 커버로스(Kerberos) ::서버와 클라이언트 간의 신원 확인 프로토콜
	- 아이핀(i-PIN) ::주민등록번호 대체

접근 통제
: 정당한 사용자에게는 권한을 부여하고 그 외의 다른 사용자는 거부하는 것
	1) 접근 통제 과정
		- 식별(Identification) : 사용자 ID를 확인하는 과정
		- 인증(Authenticaion) : 패스워드가 정확한지 확인
		- 인가(Authorization) : 읽고, 쓰고, 실행시키는 권한 부여
	2) 접근 통제 정책 ::맥댁알백
		ㄱ. 강제적 접근 통제(MAC, Mandatory Access Control) ::군대
			- 사용자 레벨과 보안 레벨을 비교하여 접근 제어
			- 기밀성이 강조되는 조직
		ㄴ. 임의적 접근 통제(DAC, Discretionary Access Control) ::친하니까 권한부여
			- 소유권을 가진 사람이 다른 사람의 접근을 허용하거나 제한
		ㄷ. 역할기반 접근 통제(RBAC, Role Based Access Control) ::팀마다의 권한
			- 사용자의 역할에 기반을 두고 접근 통제
	3) 접근 통제 모델
		ㄱ. 벨-라파둘라 모델 : 미 국방부, 기밀성, No Read Up-No Write Down
		ㄴ. 비바 모델 : 상업용, 무결성, No-Read Down-No Write Up
		ㄷ. 클락-윌슨 모델 : 상업용, 무결성
		ㄹ. 만리장성 모델 : 충돌

보안 취약점
: 불법적인 사용자의 접근을 허용할 수 있는 위협
	- 관리적 관점
	- 기술적 관점
	- 물리적 관점

보안관제 ::관제탑
: 24시간 정보자산을 지키기 위해 모니터링, 패킷 관측한다, CERT팀이 대응함
	1) 통합로그 분석 장비
		- ESM(Enterprise Security Management)
		- SOAR
		- SIEM(Security Information & Event Management)

보안 운영체제(Secure-OS), 신뢰성 운영체제(Trusted OS)
: 기존의 운영체제 내에 보안 기능을 추가한 운영체제

보안 운영체제 목적
	- 안정성
	- 신뢰성
	- 보안성

보안 솔루션
	1) 방화벽(Firewall)
		- 침입 차단 시스템
	2) 웹 방화벽(WAF, Web Application Firewall)
		- 웹서버 특화 방화벽
	3) 침입탐지시스템(IDS, Intrusion Detection System)
		- 비정상적 사용을 실시간으로 탐지하는 시스템
		- 침입탐지 방식에 따른 분류
			ㄱ. 오용탐지 : 미리 입력해 둔 패턴이 감지되면 알려준다
			ㄴ. 이상탐지 : 미리 입력해 둔 값 이외의 패턴이 감지되면 알려준다
		- 침입탐지 대상에 따른 분류
			ㄱ. 네트워크 기반 IDS(NIDS) : 패킷을 분석하여 탐지
			ㄴ. 호스트 기반 IDS(HIDS) : 로그를 분석하여 탐지
	4) 침입방지시스템(IPS, Intrusion Prevention System)
		- 방화벽 + 침입탐지시스템
	5) 데이터유출방지(DLP, Data Leakage/Loss Prevention)
		- 내부 정보의 외부 유출 방지
	6) 가상 사설 통신망(VPN, Virtual Private Network) ::재택근무
		- 회사입장에서 사용자가 마치 자신의 전용 회선을 사용하는 것처럼
		  해주는 보안 솔루션
	7) NAC(Network Access Control)
		- MAC주소를 IP관리 시스템에 등록하여 일관된 보안관리
	8) ESM(Enterprise Security Management) ::단순 이벤트
		- 발생하는 로그 및 보안 이벤트를 통합 관리
	9) SIEM(Security Information & Event Management) ::빅데이터
		- ESM과 동일하지만 빅데이터 수준 분석
	10) SOAR(Security Orchestration, Automation and Response)
		- 보안 오케스트레이션
	11) Sandbox ::놀이터, 안전한 공간
		- 원래의 운영체제와 완전히 독립되어 실행되는 형태
	12) FDX(Fraud Detection System)
		- 이상 금융거래를 차단하는 시스템
	13) Proxy Server ::대신, 대리
		- 클라이언트 대신에 인터넷상의 다른 서버에 접속하는 서버

방화벽(Firewall)
	1) DMZ 구간
		- 내부 네트워크에 포함되어 있으나 외부에서 접근할 수 있는 구간
	2) 구현방식에 따른 유형
		- 패킷 필터링 : 네트워크 계층과 전송 계층에서 동작
		- 애플리케이션 게이트웨이 : 응용계층에서 동작
		- 회선 게이트웨이 : 응용계층과 세션계층 사이에서 동작
		- 상태 기반 패킷 검사 : OSI의 모든 계층에서 패킷을 분석하여 차단
		- 혼합형 타입 : 복합적으로 구성
	3) 방화벽 시스템 구축 유형
		ㄱ. 스크리닝 라우터(Screening Router)
			- IP, TCP, UDP의 헤더 부분에 포함된 내용만 분석
		ㄴ. 베스천 호스트(Bastion Host) ::비싸서 1개만 둠
			- 진입하기 전에 베스천 호스트를 두어 내부를 전체적으로 보호
		ㄷ. 듀얼 홈드 호스트(Dual-Homed Host) ::2인터페이스짜리 베스천호스트
			- 하나의 NIC는 내부 네트워크, 다른 NIC는 외부 네트워크와 연결
		ㄹ. 스크린드 호스트(Screened Host) ::스크리닝라우터1+베스천호스트1
			- 패킷 필터 라우터와 베스천 호스트로 구성
		ㅁ. 스크린드 서브넷(Screeded Subnet) :: 스크리닝라우터2+베스천호스트1
			- 두 개의 스크리닝 라우터와 한 개의 베스천 호스트로 구성

보안 프로토콜
	1) SSH(Secure Shell Protocol) ::Telnet + 암호화
		- 원격 호스트 접속에 사용하는 보안 프로토콜, 22번 포트
	2) SSL(Secure Socket Layer) ::로그인할 때 데이터를 암호화하여 전달
		- 웹 브라우저와 웹 서버 간에 데이터를 안전하게 주고받기 위한 프로토콜
		- 적용 시 URL이 https로 시작, 443포트, http는 80포트
	3) TLS(Transport Layer Security)
		- 전송계층을 기반으로 개발
	4) IPSec
		- 네트워크계층을 보호하기 위한 기법
		- 동작모드
			ㄱ. 전송모드 : 헤더를 제외한 페이로드(데이터)만을 보호
			ㄴ. 터널모드 : 패킷 전체를 보호(헤더+페이로드)
		- 프로토콜
			ㄱ. AH(Authentication) : 무결성, 인증
			ㄴ. ESP(Encapsulation Security Payload) : 무결성, 인증, 기밀성
			ㄷ. IKE(Internet Key Exchange) : 키 교환에 사용
	5) S-HTTP(Secure HTTP) ::SSL과 비슷
		- 웹상에서 네트워크 트래픽을 암호화하는 기법
		- HTTP의 확장판
	6) RedSec
		-RADIUS

고가용성 (HA, High Availability) ::물리적으로 WAS를 2대이상 둠 (A/S)
: 서버와 네트워크, 프로그램 등의 정보 시스템이 오랜 기간 동안 지속적으로 정상 운영이 가능한 성질

DoS(Denial of Service) 공격
: 시스템이 정상적인 서비스를 할 수 없도록 가용성을 떨어뜨리는 공격

DoS 공격 유형
	1) Smurf Attack
		- IP와 ICMP의 특성 이용
	2) Ping Of Death
		- 규정 크기 이상의 ICMP 패킷으로 시스템 마비
	3) Land Attack
		- 출발지 IP와 목적지 IP가 같은 패킷 생성
	4) Teardrop Attack
		- 재조합할 수 있는 Fragment Number를 위조
	5) SYN Flooding
		- TCP의 연결과정(3Way Handshaking)의 취약점 이용
	6) UDP Flooding
		- 다량의 UDP패킷을 전송하여 자원을 고갈
	7) Ping Flooding
		- 많은 ICMP Echo를보내 자원 고갈

DDoS(Distributed Denial of Service) 공격

DDoS의 공격 구성
	1) 공격자
	2) 명령 제어
	3) 좀비 PC
	4) 공격 대상
	5) Exploit : 공격자의 의도된 명령/프로그램

DDoS 공격 툴의 종류
	1) 트리누(Trinoo) : UDP Flooding 공격 수행
	2) TFN(Tribal Flood Network) : UDP/TCP Flooding, ICMP 브로드캐스트 공격 수행
	3) 슈타첼드라트(Stacheldraht) : Trinoo + TFN

기타 해킹 기법
	1) 웜(Worm)
		- 자신을 복제하고 전파할 수 있는 악성 프로그램
	2) 바이러스(Virus) ::숙주가 있어야함
		- 스스로를 복사하는 악성 프로그램으로 파일 속에 숨어 옮겨 다님
	3) 스턱스넷(Stuxnet)
		- 공항, 발전소, 철도 등 기간시설 파괴
	4) 루팅(Rooting)
		- 핸드폰 운영체제의 루트(root) 관리자 계정 획득
	5) 루트킷(Rootkit)
		- 루트 권한을 쉽게 얻게 해주는 킷
	6) 혹스(Hoax) ::혹시..장난?
		- 가짜 바이러스
	7) 스니핑 공격(Sniffing Attack)
		- 훔쳐보는 공격
	8) IP Spoofing
		- IP주소를 속여서 접속
	9) ARP Spoofing
		- MAC을 속이는 공격
	10) DNS Spoofing ::이 공격을 이용해서 파밍을 한다
		- DNS 서버로 보내는 질문을 가로채서 변조된 결과 전달
	11) 파밍(Pharming)
		- 정상 홈페이지에 접속하여도 피싱 사이트로 유도
	12) 타이포스쿼팅(Typosquatting) ::타이핑
		- 주소 타이핑 실수를 이용하여 유사한 URL로 접속 유도
	13) Smishing(SMS phishing)
		- 문자메시지를 이용한 피싱
	14) Qshing
		- QR코드를 통해 악성 링크로 유도
	15) 포트 스캐닝(Port Scanning)
		- 서버에 열려있는 포트를 확인 후 해당 포트 공격
	16) 세션 하이재킹(Session Hijacking)
		- 이미 인증 완료된 세션의 연결을 빼앗는 공격
	17) Buffer Overflow
		- 입력받는 값이 버퍼를 넘치게하여 버퍼 이후 공간을 침범
	18) Format String Attack
		- 문자열의 출력 포맷을 애매하게 설정할 때의 취약점 포착
	19) SQL Injection
		- 입력값을 조작하여 서버의 데이터베이스를 공격
	20) XSS(Cross-Site Scripting)
		- 사이트에 스크립트를 넣어 공격
	21) CSRF(Cross-Site Request Forgery)
		- 스크립트를 넣어 특정 웹사이트에 요청하게 만드는 해킹
	22) Backdoor ::개발자들이 편하게 작업하기 위해 많이 사용
		- 정상적인 인증을 거치지 않고 시스템에 접근하는 프로그램
	23) Password Cracking
		- 비밀번호를 각종 툴을 통해 알아내는 공격
	24) Rainbow Table
		- 해시함수(MD-5, SHA)로 만들어낼 수 있는 값들을 대량 저장한 테이블
	25) APT(Advanced Persistent Threat) ::똑똑한 아파트
		- 지속적이고 지능적인 해킹 공격
	26) Nucking ::방장(king)이 강퇴시킴
		- 인터넷 접속을 끊는 크래킹
	27) 부채널 공격(Side Channel Attack)
		- 암호 알고리즘을 대상으로 한 물리적 공격
	28) Brute Force Attack
		- 무차별 대입 공격
	29) Dictionary Attack
		- 많이 사용되는 패턴들을 사전형태로 만들어 이들을 조합하여 공격
	30) Key Logger Attack
		- 키보드의 움직임 탐지하여 정보를 빼가는 공격
	31) 스파이웨어(Spyware)
		- 사용자 동의 없이 정보를 수집
	32) 애드웨어(Adware)
		- 광고를 보여줌으로써 비용 납부를 대신한다
	33) 트랙웨어(Trackware)
		- 정보를 다른 조직에 전달
	34) 그레이웨어(Grayware)
		- 일반 소프트웨어에 속하지만 설치되면 사용자에게 불편을 주는 소프트웨어
	35) 크라임웨어(Crimeware)
		- 불법 온라인 활동을 용이하게 하는 소프트웨어
	36) 랜섬웨어(Ransomware)
		- 시스템을 감염시켜 접근을 제한하고 몸값을 요구
	37) 제로데이(Zero-Day Attack)
		- 취약점에 대한 패치가 나오지 않은 시점에서 공격
	38) 사회공학(Social Engineering) ::소주마시면서 비밀알아내기
		- 신뢰를 기반으로 사람을 속여 비밀정보 획득
	39) Evil Twin Attack
		- 와이파이(무선네트워크)에서 가짜 AP에 접속하게 하는 공격
	40) Bluebug
		- 한 번 연결되면 자동으로 연결되는 취약점 이용
	41) Bluesnarf
		- 파일에 접근하는 공격
	42) BluePrinting
		- 검색 활동
	43) BlueJacking
		- 익명메시지를 보내는 해킹 방법
	44) Switch Jamming ::멍청하게 만드는 공격
		- 스위칭허브를 더미허브처럼 작동하게 하는 공격
	45) 블루킵(Bluekeep) ::윈도우 원격
		- 원격 데스크톱 서비스를 인증 없이 조작
	46) 인포데믹스(Infodemics)
		- 잘못된 정보나 소문이 확산되어 대중의 두려움 증폭
	47) 다크 데이터(Dark Data)
		- 저장만 하고 활용하고 있지 않은 데이터
	48) 킬 스위치(Kill Switch)
		- 일종의 자폭
	49) 트러스트존(TrustZone) ::신뢰존
		- 독립적인 보안 구역을 따로 두어 중요한 정보를 보호
