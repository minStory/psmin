01 프로그래밍 언어 기초

알고리즘 표현 방법
	1) 자연어
		- 우리가 사용하는 언어로 알고리즘 표현
	2) 의사코드(Pseudo Code)
		- 프로그램 코드를 흉내내어 알고리즘 작성
	3) 순서도(FLow Chart)
		- 기호와 도형을 사용하여 일의 흐름 표현
	4) 나씨 슈나이더만 차트(Nassi-Schneiderman Chart)
		- 순차, 선택, 반복의 구조를 사각형으로 도식화하여 알고리즘 표현

변수명 표기법
	1) 카멜 표기법(Camel Case)
		- 앞 단어를 제외한 첫 글자를 대문자로 표기
		- parkSeongMin
	2) 파스칼 표기법(Pascal Case)
		- 모든 단어의 앞자리를 대문자로 표기
		- ParkSeongMin
	3) 스네이크 표기법(Snake Case)
		- 단어와 단어 사이에 언더바로 표기
		- park_seong_min
	4) 헝가리안 표기법
		- 접두사에 자료형을 붙여서 표기
		- strName
		- intAge

프로그래밍 언어의 유형 분류

개발 편의성 측면에 따른 분류
	1) 저급 언어(Low-Level Language)
		- 컴퓨터가 직접 이해할 수 있는 언어
	2) 고급 언어(High-Level Language)
		- 인간이 이해할 수 있는 소스코드로 되어 있는 언어

실행 및 구현 방식에 따른 분류
	1) 명령형 언어(Imperative Language)
	2) 함수형 언어(Functional Language)
	3) 논리형 언어(Logic Language)
	4) 객체지향 언어(Object-Oriented Language)

빌드 방식에 따른 분류
	1) 컴파일 언어(Compile Language)
		- 소스코드를 목적코드로 변환하여 실행
	2) 인터프리터 언어(Interpreter Language)
		- 소스코드를 한 줄 씩 실행
	3) 바이트 코드 언어(Byte Code Language) ::Java의 JVM
		- 가상머신이 번역할 수 있는 Byte Code로 변환, 가상머신이 기계어로 번역

절차적 프로그래밍 언어
: 처리 절차를 정해진 문법에 따라 순서대로 기술하는 언어

절차적 프로그래밍 언어 종류
	- C언어
	- COBOL
	- FORTRAN
	- BASIC

객체지향 프로그래밍 언어
: 구조를 객체로 만들고, 객체들끼리 상호작용하도록 만드는 프로그래밍 언어

객체지향 프로그래밍 특징
	- 캡슐화(Encapsulation)
	- 정보은닉(Information Hiding)
	- 상속(Inheritance)
	- 다형성(Polymorphism)
	- 추상화(Abstraction)

객체지향 프로그래밍 언어의 종류
	- JAVA
	- 시뮬라67
	- 스몰토크(Smalltalk)
	- 오브젝티브-C(Objective-C)
	- C++
	- 파이썬(Python)

스크립트 언어 ::끼워넣는 언어
: 응용 소프트웨어를 제어하는 컴퓨터 프로그래밍 언어

스크립트 언어의 종류
	- Java Script
	- JSP
	- PHP
	- ASP
	- Perl
	- Python
	- VBScript

선언형 언어 ::어떤 데이터를 가지고 오라는 결과만 명시
: 프로그램이 수행해야 하는 문제를 기술하는 언어

선언형 언어의 종류
	- 하스켈(Haskell)
	- HTML
	- SQL
